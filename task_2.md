**Верхнеуровневая постановка (User Story)**

Как продавец я хочу публиковать свои товары на витрине маркетплейса, чтобы:
1.	Находить новых покупателей через централизованную платформу;
2.	Увеличить продажи за счет расширения аудитории;
3.	Автоматизировать процесс создания карточек товаров;
4.	Контролировать видимость своих товаров (публиковать/скрывать);
5.	Быстро обновлять информацию о товарах (цены, остатки, описания)

Критерии приемки (Acceptance Criteria):

•	Продавец может создать новую карточку товара через Личный кабинет;

•	Система проводит базовую валидацию обязательных полей;

•	Товар отправляется на модерацию перед публикацией;

•	Продавец видит понятный статус публикации (в обработке/одобрено/отклонено);

•	После одобрения товар появляется на витрине маркетплейса;

•	Продавец может скрыть опубликованный товар в любой момент;

•	Изменения в карточке товара требуют повторной модерации

**Варианты использования (Use Case)**

**1. Создание карточки товара**

*Условие:* Продавец авторизован в ЛК

1.	Продавец переходит в раздел "Мои товары" → "Добавить товар"

2.	Система отображает форму создания товара

3.	Продавец заполняет обязательные поля:

•	Название товара (например до 100 символов)

•	Категория (выбор из каталога маркетплейса)

•	Цена (с указанием валюты)

•	Количество на складе

•	Фотографии товара (минимум 1, максимум 12)

4.	Продавец заполняет необязательные поля:

•	Описание товара

•	Характеристики (вес, габариты, материал и др.)

•	Артикул продавца

•	Производитель

•	Страна производства

5.	Продавец нажимает "Сохранить и отправить на модерацию"

6.	Система валидирует данные:

•	Проверяет заполнение обязательных полей

•	Проверяет корректность форматов (цена > 0, количество ≥ 0)

•	Проверяет размер и формат фотографий

7.	Система сохраняет карточку товара со статусом "На модерации"

8.	Система показывает уведомление: "Товар отправлен на модерацию. Среднее время проверки — 24 часа"

**2. Редактирование карточки товара**

*Условие:* Продавец авторизован и товар уже в системе

1.	Продавец переходит в список товаров → выбирает товар → нажимает "Редактировать"

2.	Система открывает форму редактирования

3.	Продавец вносит изменения

4.	Продавец нажимает "Сохранить изменения"

5.	Если товар был опубликован:

•	Система меняет статус на "На модерации"

•	Товар временно скрывается с витрины

•	Система уведомляет: "Изменения отправлены на повторную проверку"

6.	Если товар был в черновике или отклонён:

•	Система сохраняет изменения без смены статуса

•	Товар остаётся неопубликованным.

**3. Управление видимостью товара**

*Условие:* Товар одобрен модерацией и опубликован

*Основной поток (Скрыть товар):*

1.	Продавец переходит в список товаров → выбирает опубликованный товар

2.	Продавец нажимает "Скрыть с витрины"

3.	Система меняет статус видимости на "Скрыт"

4.	Товар пропадает с витрины маркетплейса

5.	Продавец видит статус "Скрыт" в Личном кабинете.

*Альтернативные потоки:*

1.	Вернуть на витрину

2.	Для скрытого товара продавец нажимает "Вернуть на витрину"

3.	Система мгновенно меняет статус на "Опубликован"

4.	Товар снова появляется на витрине.

**4. Дополнительные требования и ограничения**

Например, следующие **бизнес-правила:**

1.	Модерация - обязательна для первой публикации товара и для изменений в ключевых полях (название, категория, цена, фото)

2.	Изменение остатков не требует повторной модерации (синхронизация в реальном времени)

3.	Лимиты:

•	Максимум 500 активных товаров у одного продавца (без верификации)

•	Максимум 12 фото на товар, размер до 10 МБ каждая

•	Цена товара должна быть в диапазоне 10 - 1 000 000 руб

4.	Запрещённый контент: товары под запретом (оружие, наркотики и т.п.) отклоняются без возможности повторной публикации

Например, следующие **интеграционные требования:** 

1.	API для синхронизации остатков с внешними складскими системами

2.	Webhook-уведомления о смене статуса модерации.
